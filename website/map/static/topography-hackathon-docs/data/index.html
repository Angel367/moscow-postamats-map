Методы обработки данных

Для работы решения требуются большие количества данных. В данном разделе описаны их источники и методы обработки.

Население

Датасет, прикрепленный к ТЗ, содержал в себе слишком малое кол-во данных, поэтому было принято решение воспользоваться альтернативным источником.

Среди альтернатив выбор состоял из Yandex Maps, 2gis и OpenStreetMap.

API Yandex Maps и 2gis имели много ограничений и в некоторых местах уступали датасету по кол-ву информации, поэтому выбор пал на OpenStreetMap и инстанс Overpass API от VK, который не предусматривает ограничений.

В OpenStreetMap, в большинстве случаев, содержится точная информация о всех строениях Москвы, которую несложно отсортировать с помощью Overpass API QL.

С помощью данных Росстата о населении Москвы по районам и интерполяции было произведено распределение строений по районам, затем населения по зданиям в районе.

Районы и округа

Границы районов и округов а также их названия были взяты в формате GeoJSON из OpenStreetMap.

Иные постаматы

Yandex Maps содержит в себе данные о практически всех уже существующих постаматов в Москве поэтому их расположение было взято оттуда.

В связи с ограничениями Yandex Maps было необходимо разделить Москву на сектора и запросить данные для каждого сектора отдельно с последующим объединением в единый датасет.

Важные городские объекты

С портала открытых данных Правительства Москвы были использованы следующие данные:

  • Нестационарные торговые объекты по реализации печатной продукции
  • Нестационарные торговые объекты
  • Многофункциональные центры предоставления государственных и муниципальных услуг
  • Библиотеки города
  • Дома культуры и клубы
  • Спортивные объекты города Москвы
